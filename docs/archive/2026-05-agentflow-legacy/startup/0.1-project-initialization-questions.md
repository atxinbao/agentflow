## 目的

0.1 先确定项目，不写代码、不做蓝图。以下答案来自既有讨论，状态为 `inferred`。

| # | Question | Current Answer |
| --- | --- | --- |
| 1 | 一句话是什么？ | 闭源、免费、本地优先的 AI 研发执行工具。 |
| 2 | 给谁用？ | 独立开发者、2-10 人小团队、小型外包团队、中国开发者。 |
| 3 | 痛点是什么？ | AI 写代码缺少边界、验证、审查和本地可控性。 |
| 4 | 为什么不是 PM 平台？ | 只做 AI 本地执行闭环，不做完整项目管理。 |
| 5 | 第一原则？ | 本地优先，代码默认不上传。 |
| 6 | 是否开源？ | 本体闭源，数据格式开放。 |
| 7 | 免费边界？ | Desktop Free 覆盖单机本地核心闭环。 |
| 8 | 第一版闭环？ | repo -> issue contract -> run -> validation -> evidence -> update -> review。 |
| 9 | 明确不做？ | SaaS、账号、支付、云同步、完整移动执行端、自动 merge、海外代理。 |
| 10 | 技术路线？ | Rust Core/CLI + Tauri + React/TS + Markdown/JSON + SQLite。 |
| 11 | 模型策略？ | OpenAI-compatible first，兼容国内和本地模型。 |
| 12 | 数据在哪里？ | `.agentflow/` 是事实源，SQLite 是可重建索引。 |
| 13 | 什么是可执行任务？ | 含 scope、non-goals、context、validation、evidence requirements 的 issue contract。 |
| 14 | 商业路径？ | 免费积累用户，后续 Local Pro、Team Sync、Enterprise Local 或被收购。 |
| 15 | 第一施工包？ | Goal Compiler + Core/CLI Bootstrap v0。 |
