创建日期：2026-05-21
最近压缩：2026-05-22
执行者：Codex

## 战略

闭源免费 + 本地优先 + 数据开放。

| 选择 | 原因 |
| --- | --- |
| 免费 | 降低分发门槛，先积累个人和小团队用户 |
| 闭源 | 保留 Pro / Team / Enterprise / 被收购空间 |
| 本地优先 | 避开上传代码、海外账号、支付和合规阻力 |
| 数据开放 | 降低试用顾虑，不等于开源核心 |

## 免费边界

Desktop Free 永久覆盖：

- 单机本地项目。
- issue contract。
- basic run / validation。
- evidence report。
- project update。
- review / PR 文案。

延后收费能力：Team Sync、Private Deployment、Model Gateway、高级模板、高级 evidence 分析、企业集成。

## 路线

| 路线 | 资产 |
| --- | --- |
| 被收购 | 用户量、桌面体验、本地执行协议、模型适配、`.agentflow/` 工作流 |
| 小型商业化 | Local Pro、Team Sync、Enterprise Local、模板/插件包、咨询落地 |

## 竞争点

不和 Linear、PingCode、ONES、TAPD、飞书项目竞争完整 PM 能力。

只打一个点：本地 AI Agent 执行边界 + 验证 + evidence。

## 验证问题

- 用户当前用什么 AI 写代码？
- 最大问题是质量、上下文、测试，还是边界？
- 是否愿意安装本地工具管理 AI 执行？
- 是否接受闭源免费但数据开放？
- 需要哪些模型预设？
